# Flight Management System

A relational database project built to demonstrates the use of Python and SQLite to manage
essential airline operations including flights, pilots, passengers,
and bookings.It covers database design, data interaction and query execution through a
command-line interface.

---

## Project Structure

```
flightmanagement_repolink/
│
├── airline_data_management.py   # Single script — tables, data and CLI
├── airline_database.db          # SQLite database (auto-created on first run)
└── README.md
```

---

## Database Overview

The database is called `airline_database.db` and contains 6 tables.

### Tables

| Table | Description |
|---|---|
| `airports` | Stores all airports the airline operates routes to and from |
| `aircraft` | The airline's fleet of aircraft |
| `pilots` | Pilot records including licence numbers |
| `flights` | Central table linking aircraft, pilots and airports |
| `passengers` | Passenger identity and contact details |
| `bookings` | Booking transactions linking passengers to flights |

### Relationships

| Relationship | Type | Description |
|---|---|---|
| airports → flights | One to Many | One airport serves many flights (departure and arrival) |
| aircraft → flights | One to Many | One aircraft operates many flights |
| pilots → flights | One to Many | One pilot is assigned to many flights |
| flights → bookings | One to Many | One flight has many bookings |
| passengers → bookings | One to Many | One passenger makes many bookings |

## How to Run

**Step 1 — Run the script**

```bash
python3 airline_data_management.py
```

On the first run, the script automatically:
- Creates the `airline_database.db` file
- Creates all 6 tables
- Inserts all sample data

Every run after that goes straight to the main menu.

**Step 2 — Use the menu**

```
========================================
      FLIGHT MANAGEMENT SYSTEM
========================================
  1. Manage Flights
  2. Manage Pilots
  3. Manage Bookings
  4. Manage Destinations
  5. Summary Reports
  0. Exit
```

---

## Menu Options

### 1. Manage Flights
| Option | Description |
|---|---|
| View all flights | Lists all flights with route, times and status |
| Search flight by ID | Retrieves a single flight record |
| Add a new flight | Inserts a new flight with aircraft, pilot and route |
| Update departure time | Changes the departure datetime of a flight |
| Update flight status | Changes status to scheduled, departed, landed, delayed or cancelled |
| Delete a flight | Removes a flight record by ID |

### 2. Manage Pilots
| Option | Description |
|---|---|
| View all pilots | Lists all pilots with licence numbers |
| Assign pilot to flight | Updates a flight record with a selected pilot |
| View pilot schedule | Shows all flights assigned to a specific pilot |

### 3. Manage Bookings
| Option | Description |
|---|---|
| View all bookings | Lists all bookings with passenger and flight details |
| Add a new booking | Creates a booking linking a passenger to a flight |
| Delete a booking | Removes a booking record by ID |

### 4. Manage Destinations
| Option | Description |
|---|---|
| View all destinations | Lists all airports with IATA codes, city and country |
| Update airport information | Updates city or country details for an airport |

### 5. Summary Reports
| Option | Description |
|---|---|
| Flights per destination | Number of flights arriving at each airport |
| Flights per pilot | Number of flights assigned to each pilot |
| Passengers per destination | Number of bookings per arrival airport |
| Flight duration by route | Average flight time in hours per route |
| Flights by departure date | Number of flights on each departure date |

## SQL Concepts Demonstrated

| Concept | Where Used |
|---|---|
| `CREATE TABLE` | All 6 tables created on first run |
| `INSERT INTO` | Sample data insertion, Synthetic data generated by AI |
| `SELECT` with `JOIN` | Flight views, bookings, pilot schedules |
| `WHERE` | Search by ID, filter by status or date |
| `UPDATE` | Departure time, flight status, pilot assignment, airport info |
| `DELETE` | Remove flight or booking records |
| `GROUP BY` + `COUNT` | Summary reports |
| `AVG` + `JULIANDAY` | Average flight duration calculation |
| `UNIQUE` constraint | iata_code, passport_number, email, licence_number |
| `AUTOINCREMENT` | Primary keys on all tables |

## Future Extensions

The following areas were intentionally kept out of scope for this
represent natural next steps:

- Crew assignments, linking multiple staff members to each flight
- Pilot licences with a separate table for license types like ATPL, CPL and PPL records
- Flight delays by logging individual delay events per flight
- Codeshare and partner airlines for inter airline agreements
- Fare amounts and payment tracking
- Gate and terminal management
